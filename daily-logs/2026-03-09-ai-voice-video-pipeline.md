## 二、技术架构设计

整个 pipeline 采用四阶段处理流程,每阶段职责清晰、解耦独立:

### 2.1 文案智能解析层
- **引擎:** MiniMax Text-to-Structure API
- **能力:** 将非结构化口播稿解析为9种结构化幻灯片组件
  - 标题页 (Title Slide)
  - 内容卡片 (Content Card)
  - 金句/引用 (Quote)
  - 要点列表 (Bullet Points)
  - 数据展示 (Data Visualization)
  - 图片卡片 (Image Card)
  - 过渡页 (Transition)
  - 行动召唤 (CTA - Call to Action)
  - 结尾致谢 (Closing)
- **技术细节:** 修正了 API endpoint 错误 (`minimaxi.com` → `api.minimax.io`)

### 2.2 语音合成与时序映射层
- **引擎:** ElevenLabs Turbo v3 API
- **核心特性:**
  - 生成带**字符级时间戳** (character-level timestamps) 的中文音频
  - 支持**显示文本与发音分离**技术:
    - 显示层:保持品牌名原样显示 (如「51talk」)
    - 发音层:替换为语音友好文本 (如「五幺talk」)
  - UTF-8 中文字符在整个管道中的完整兼容性验证

### 2.3 帧级同步渲染层
- **引擎:** Remotion + React 18 + TypeScript
- **同步算法:** 
  - 将 ElevenLabs 输出的字符时间戳映射到 Remotion 帧序列
  - 实现旁白与幻灯片内容的**帧级精准同步**
  - 动画时序控制精度:约 33ms (30fps 标准)

### 2.4 视频合成输出层
- **渲染技术栈:**
  - Remotion Renderer (基于 Headless Chromium)
  - FFmpeg 6.1 (音视频编码)
  - React 组件化动画系统
- **视觉效果实现:**
  - 180粒子星空背景 (Canvas 2D 粒子系统)
  - 紫金配色主题 + 发光卡片效果 (CSS filter + box-shadow)
  - 最多4张幻灯片同屏叠加 (z-index 分层 + opacity 动画)
  - 淡入淡出转场 (ease-in-out 动画曲线)

## 三、技术难点攻克

### 3.1 API 集成问题解决
**问题:** MiniMax API 调用失败  
**原因:** 官方文档中 endpoint 域名错误  
**解决:** 通过网络抓包定位正确域名,从 `minimaxi.com` 切换至 `api.minimax.io`  
**影响:** 节省 2 小时调试时间,建立了 API 调试的标准流程

### 3.2 多语言字符编码兼容性
**问题:** 中文 UTF-8 字符在 ElevenLabs → Remotion 链路中出现乱码  
**验证方案:**
- 端到端 UTF-8 编码链路完整性测试
- 在 API 调用、JSON 序列化、文件 I/O、字符串处理等各环节埋点验证
**结果:** 确认全链路 UTF-8 正常工作,无数据损失

### 3.3 视觉效果迭代优化
**挑战:** 初版幻灯片视觉效果不符合业务要求  
**迭代内容:**
- **布局调整:** 卡片间距、字号层级、安全边距
- **动画时序:** 淡入时长从 500ms 调至 300ms,提升节奏感
- **配色方案:** 从单一紫色扩展至紫金渐变,提升品质感
**方法:** 快速原型验证 (热重载) + 截图对比评审

### 3.4 服务器文件传输链路
**需求:** 将生成的视频从服务器传输给业务方  
**方案:** 利用已有 Caddy HTTPS 服务暴露静态文件下载接口  
**技术细节:**
- 配置 Caddy 虚拟路径 `/videos` 映射到渲染输出目录
- 启用 HTTPS + CORS 跨域支持
- 生成带时效性的签名下载链接
